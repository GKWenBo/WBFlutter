# M8 登录与鉴权（Form / secure storage / redirect）⭐

> 本课产出：登录流程 + 全局鉴权拦截——`Form` 表单校验、DummyJSON auth、
> token 存 `flutter_secure_storage`、dio 自动带 token、go_router `redirect` 拦未登录访问。
> **这是 iOS 背景的杀手锏模块：Keychain、拦截器加 header、Coordinator 拦截，你都有对位。
> 本课把 M3 的拦截器、M5 的路由表、M7 的 keepAlive Notifier 三条线拧成一股。**

---

## 一、本课重点掌握（按重要程度排序）

### 1. `redirect` 集中拦截：鉴权不散落在各页面 ⭐⭐⭐

```dart
redirect: (context, state) {
  final isLoggedIn = ref.read(authProvider).asData?.value != null;
  final needsAuth = _protectedPaths.any((p) => state.matchedLocation.startsWith(p));
  if (!isLoggedIn && needsAuth) return '/login';   // 未登录闯鉴权页 → 拦到登录
  if (isLoggedIn && state.matchedLocation == '/login') return '/home'; // 已登录别再看登录页
  return null;                                       // 放行
};
const _protectedPaths = ['/profile', '/checkout'];   // 加一个字符串就多保护一个页面
```

- **一处 redirect 管全局鉴权**（≈ Coordinator 在导航前统一判断），而不是每个页面 `initState` 里自己查登录。
- 回报在 M10 兑现：结算要登录，只往 `_protectedPaths` 加 `'/checkout'`，**拦截逻辑一行没动**。
- `redirect` 里用 `ref.read` 而非 `watch`：它是一次性回调、不是 build 方法（见第 2 点为何仍能实时生效）。

### 2. `refreshListenable`：把 Riverpod 状态变化"喂给"go_router ⭐⭐⭐

```dart
class _AuthRefreshListenable extends ChangeNotifier {
  _AuthRefreshListenable(Ref ref) { ref.listen(authProvider, (_, _) => notifyListeners()); }
}
// GoRouter(refreshListenable: _AuthRefreshListenable(ref), redirect: ...)
```

- 问题：`redirect` 只在"发生一次导航"时被调用。但**登出**这种状态变化不来自导航——
  用户还停在"我的"页只是点了个按钮，没人触发 redirect 去把他踢到登录页。
- 解法：`refreshListenable` 是个桥。`ref.listen(authProvider)` 监听登录态，一变就 `notifyListeners()`，
  go_router 收到后**主动重跑一遍 redirect**——于是登出瞬间"我的"页被拦回登录。
- iOS 对照：≈ 用 `NotificationCenter`/Combine 通知 Coordinator"登录态变了，重新决定该展示哪页"。

### 3. token 存 Keychain（`flutter_secure_storage`），不存 UserDefaults ⭐⭐⭐

```dart
class AuthStorage {   // 放 core/storage/，因为 core/network 的拦截器和 auth feature 都要用
  final _storage = const FlutterSecureStorage();
  Future<String?> readAccessToken() => _storage.read(key: _accessTokenKey);
  Future<void> saveTokens({required accessToken, required refreshToken}) async {...}
  Future<void> clear() async {...}
}
```

- token 是敏感凭证，**明文存 shared_preferences/UserDefaults 不安全** → 换 `flutter_secure_storage`，
  iOS 端底层就是 **Keychain**。对比 M7 的 CartStorage（普通数据用 UserDefaults 就行）。
- 放 `core/storage/` 而非 `features/auth/data/`：因为它**被多方共用**——`AuthInterceptor`（读 token 拼 header）
  + auth Provider（登录写、登出清）。"被多方共用"是它下沉到 core 的判据。

### 4. `AuthInterceptor` 自动带 token：一处加、全局请求都带 ⭐⭐

```dart
void onRequest(options, handler) async {
  final token = await _storage.readAccessToken();
  if (token != null) options.headers['Authorization'] = 'Bearer $token';
  handler.next(options);
}
```

- 呼应 M3：拦截器的 `onRequest` **可以是 async**——dio 会等到 `handler.next()` 才发请求，
  所以"异步读一次 Keychain 再决定加不加 header"能成立。加在 DioClient 里一次，所有请求自动带 token。

### 5. 冷启动自动登录 + token 失效自愈 ⭐⭐

```dart
Future<User?> build() async {
  final token = await ref.read(authStorageProvider).readAccessToken();  // 启动查一次 Keychain
  if (token == null) return null;
  try { return await ref.read(authRepositoryProvider).fetchCurrentUser(); }
  catch (_) { await ref.read(authStorageProvider).clear(); return null; } // 失效→清凭证→回未登录
}
```

- 冷启动 `build()` 查一次本地 token 决定要不要自动登录（≈ iOS 启动查 Keychain）。
- token 过期/失效：**清掉本地凭证、回退未登录**，而不是一直卡在错误页。这是"自愈"，体验关键。
- 读取本身也 try：存储损坏就当未登录，别让 App 崩。

### 6. login/logout 是"整替换"，不需要 `await future` ⭐⭐

- 对比 M7：Cart 加购要在旧列表上改一部分 → 先 `await future`；
  login 成功就是新用户、logout 就是 null，**不依赖旧状态** → 直接 `state = ...`，不用先读旧值。
- `login` 用 `state = await AsyncValue.guard(() async {...})`：`guard` 自动把成功包 `AsyncData`、
  异常包 `AsyncError`——省手写 try/catch 转三态。

### 7. `Form` + `TextFormField` 校验 ⭐⭐

```dart
final _formKey = GlobalKey<FormState>();               // ≈ 校验器句柄
// 提交时：if (!_formKey.currentState!.validate()) return;  跑一遍所有 validator
TextFormField(validator: (v) => (v==null||v.trim().isEmpty) ? '请输入用户名' : null)
```

- `validator` 返回**非 null = 错误提示文案，null = 通过**。`_formKey.currentState.validate()` 一次触发全部。
- 提交按钮：`onPressed: isSubmitting ? null : _submit`——`onPressed: null` 就是禁用态（自动变灰），
  提交中防重复点。`obscureText: true` ≈ `isSecureTextEntry`。

## 二、新控件/API 速查表

| 概念/API | iOS 类比 | 怎么用 | 坑 |
|---|---|---|---|
| go_router `redirect` | Coordinator 导航前拦截 | 返回目标 path 拦截、返回 null 放行 | 用 `ref.read` 不 watch；靠 refreshListenable 触发重跑 |
| `refreshListenable` | NotificationCenter/Combine 通知 | `ChangeNotifier` + `ref.listen` → `notifyListeners` | 登出等"非导航的状态变化"必须靠它才生效；记得 dispose |
| `flutter_secure_storage` | Keychain | `read`/`write`/`delete` | 存敏感凭证；别用 shared_preferences 存 token |
| `AuthInterceptor`（async onRequest） | 统一加 Authorization header | `onRequest` 里读 token 拼 header | 别忘 `handler.next()`；异步读 Keychain 时 dio 会等 |
| `AsyncValue.guard` | try/catch 转态 | `state = await AsyncValue.guard(() async {...})` | 成功/异常自动包 Data/Error，别再手写 try |
| `Form` + `GlobalKey<FormState>` | 表单 + 校验句柄 | `validate()` 一次跑全部 validator | key 别复用；validate 只校验，不提交 |
| `TextFormField.validator` | 输入校验 | 返回非 null=错误文案 | 返回 '' 空串也算"有错"（要么 null 要么有内容） |
| `onPressed: null` = 禁用 | `isEnabled = false` | 提交中传 null 防重复点 | 写 `() {}` 看着能点其实没反应，用户困惑 |
