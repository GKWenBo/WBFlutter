# M10 下单结算（mock 闭环）

> 本课产出：购物车「去结算」→ 结算页（收货地址表单 + 商品清单）→ 提交订单 →
> 订单列表（可展开明细）。DummyJSON 没有真实下单接口，订单本地生成、本地持久化，
> 但**模型、分层、流程全部按真实项目的样子搭**——以后接真后端只换"生成订单"这一步。

---

## 一、本课重点掌握（按重要程度排序）

### 1. 流程编排放在 UI 层，不让 Service 互相调用 ⭐⭐⭐

下单成功后要连做四件事：**下单 → 存默认地址 → 清空购物车 → 跳转订单列表**。
这个串联写在 `CheckoutPage._submit()` 里，而**不是**写在 `Orders.placeOrder()` 里。

- 为什么：`placeOrder` 只管订单自己的事。如果让它反手去 `ref.read(cartProvider.notifier).clear()`，
  orders 就依赖了 cart 的**状态管理**，两个 feature 从"单向依赖模型"变成"互相牵连"，
  以后想单独测/单独改哪个都难。
- iOS 对照：Coordinator/调用方串联多个 Service，而不是让 OrderService 去改 CartService 的状态。
- 记忆钩子：**provider 方法管"一件事"，页面回调管"一串事"**。

### 2. 订单行是"快照的快照"：用独立类型表达不可变语义 ⭐⭐⭐

`OrderItem` 和 `CartItem` 字段几乎一样，为什么不复用？**语义不同**：
购物车行是可变的（数量随时改），订单行是下单那一刻的永久冻结。
`OrderItem.fromCartItem()` 工厂就是"冻结"动作。

- 依赖方向：orders → cart（只 import 它的**模型**），cart 永远不知道 orders 存在。
- iOS 对照：草稿模型 vs 落库模型分开定义，宁可小重复，不要错误共享。
- 呼应 M9：收藏存 Product 快照也是同一思路（快照换离线可用/历史不变）。

### 3. 嵌套对象序列化：`explicitToJson: true` ⭐⭐

`Order` 里嵌着 `List<OrderItem>` 和 `ShippingAddress`。json_serializable 默认的
`toJson()` 是**浅转换**——嵌套字段留着对象引用不递归。加上
`@JsonSerializable(explicitToJson: true)` 才会逐层调用子对象的 `toJson()`。

- `jsonEncode` 的兜底机制（遇到不认识的对象会试着调 `.toJson()`）经常能"碰巧救回来"，
  所以这个坑**平时不报错、换个写法就炸**——显式写进注解最稳。
- 顺带掌握：`DateTime` 自动转 ISO8601 字符串；枚举配 `@JsonValue('xxx')` 存字符串
  （**别存下标**：往枚举中间插新值时老数据全部错位）。

### 4. Dart 增强枚举（enhanced enum）⭐⭐

```dart
enum OrderStatus {
  @JsonValue('submitted') submitted('已提交'),
  @JsonValue('paid')      paid('已支付'),
  @JsonValue('shipped')   shipped('已发货');

  final String label;
  const OrderStatus(this.label);
}
```

枚举自带字段和构造函数 ≈ Swift enum 的 rawValue/计算属性。
UI 直接用 `order.status.label`，不用在页面里写 switch 转文案——文案跟着枚举走，加状态改一处。

### 5. 表单提交的完整姿势（复用 M8 + 三个新点）⭐⭐

- **防重复提交**：`_submitting` 标志 + `onPressed: _submitting ? null : _submit`。
  Material 里 **`onPressed: null` 就是禁用态**（自动变灰）——这是 Flutter 的惯用法，
  不像 iOS 有单独的 `isEnabled` 属性。
- **异步预填**：上次地址读回来后塞进 `TextEditingController`，要用**一次性**的
  `ref.read(xxx.future).then(...)`；如果在 build 里 watch 完直接 `controller.text = ...`，
  每次重建都会覆盖用户正在编辑的内容。
- **`mounted` 检查**：每个 `await` 之后要用 `context`/`setState` 前先 `if (!mounted) return`
  （≈ 闭包里的 `weak self` 判空）。用户可能在请求期间退出页面。

### 6. 路由细节：pushReplacement + 鉴权表的兑现 ⭐

- 下单成功跳订单列表用 `context.pushReplacement('/orders')`：订单列表**顶替**结算页，
  返回时回到购物车，而不是回到一个已经没意义的结算页。
  （对照：`push` 摞一层、`go` 重算整栈、`pushReplacement` 换当前层。）
- `/checkout` 加进 `_protectedPaths` 就完成了"结算必须登录"——M8 设计 redirect 时
  留的扩展点，今天只加了一个字符串，拦截逻辑一行没动。**这就是当初把鉴权做成
  "路径表 + 统一 redirect"而不是散在各页面判断的回报。**

## 二、新控件速查表

| 控件/概念 | iOS 类比 | 怎么用 | 坑 |
|---|---|---|---|
| `ExpansionTile` | UITableView 可折叠 section（iOS 要自己维护 isExpanded + insert/deleteRows） | `title`/`subtitle` 是收起态，`children` 是展开内容；自带箭头旋转动画，展开状态自己管 | 展开区域默认没有 padding，用 `childrenPadding` 补；放 Card 里配 `clipBehavior: Clip.antiAlias` 防止水波纹溢出圆角 |
| 增强枚举 enum with fields | Swift enum + rawValue/计算属性 | 枚举值带构造参数：`submitted('已提交')`；字段声明在枚举值之后 | 所有枚举值必须走同一个 const 构造函数；序列化配 `@JsonValue` 存字符串别存下标 |
| `@JsonSerializable(explicitToJson: true)` | Codable 自动递归（Swift 没这坑） | 模型含嵌套对象/对象列表时必加 | 不加时 toJson 浅转换；jsonEncode 兜底常能掩盖问题，显式声明最稳 |
| `onPressed: null` = 禁用态 | `button.isEnabled = false` | 传 null 而不是空闭包，按钮自动变灰不可点 | 写成 `onPressed: () {}` 按钮看着能点实际没反应，用户困惑；禁用就传 null |
| `textInputAction` | UIReturnKeyType | `TextInputAction.next` 让键盘回车键变"下一项"，自动移焦点 | 最后一个输入框该用 `done`；多行输入（maxLines>1）回车是换行，next 无效 |
| `TextInputType.phone` | UIKeyboardType.phonePad | `keyboardType:` 参数控制弹什么键盘 | 只是换键盘**不做校验**，校验仍靠 validator（用户可以粘贴任意内容进来） |

## 五、练习

给订单列表加一个「再次购买」按钮（ExpansionTile 展开区底部）：
把该订单的所有 `OrderItem` 逆向转回购物车（调 `cartProvider.notifier.addProduct` 不合适——
它要 `Product`；想想应该给 Cart 加什么方法，注意依赖方向：谁 import 谁？）。
做完你会对"orders → cart 单向依赖"有更深的体感。
