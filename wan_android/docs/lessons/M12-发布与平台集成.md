# M12 发布 + 平台集成（多环境 Flavor + 国际化 i18n）

> 本课把 App 从"能跑"推向"能上架"：一套代码要能切 dev/staging/prod 三套环境，
> 界面要能中英双语。**原生通道（MethodChannel/Pigeon）不在本课**——你另有 NativeLab
> 整条 L 线在系统地讲原生交互，这里不重复；本课聚焦纯 Dart 层就能搞定、且能稳定验证的
> 两块工程能力。

## 一、本课重点掌握

### 1. 多环境 Flavor：用 `--dart-define` 做编译期配置 ⭐⭐⭐

iOS 对照：≈ Xcode 的 **Build Configuration + Scheme + .xcconfig**，以及 `#if DEBUG`。
你在 iOS 里用不同 Scheme 切 API 域名、Bundle Id、日志开关；Flutter 这里用**编译期常量**
`--dart-define` 做同一件事，好处是**不动原生工程、纯 Dart 一层搞定**。

```dart
// core/config/app_env.dart
static const String _rawFlavor = String.fromEnvironment('FLAVOR', defaultValue: 'dev');
static final FlavorConfig current = flavorConfigFor(parseFlavor(_rawFlavor));
```

```bash
flutter run                                  # 默认 dev
flutter run --dart-define=FLAVOR=staging     # 预发
flutter build apk --dart-define=FLAVOR=prod  # 生产打包同理
```

关键设计点：

- **为什么用 `String.fromEnvironment` 而不是运行时读配置文件**：它是**编译期常量**——
  只认编译时 `--dart-define` 传进来的值，会被 tree-shaking，且运行时改不了。
  "选哪套环境"本就该在打包那一刻定死，不该运行时可变。
- **解析/映射抽成纯函数**（`parseFlavor` / `flavorConfigFor`）：因为 `AppEnv.current`
  依赖编译期常量、测试里改不了，把逻辑抽成纯函数才能单测（见 `test/app_env_test.dart`）。
- **兜底 dev 而不是 prod**：`parseFlavor` 认不出的值一律当 dev。宁可本地误连测试库，
  也绝不能因为拼错环境名把开发包连上生产库。这是"安全默认"的思路。
- 落地的三处差异：`DioClient` 的 **baseUrl**、是否挂 **LoggingInterceptor**、
  以及 `app.dart` 里按环境盖的 **红色 DEV/STAGING 角标水印**（prod 不显示，
  ≈ iOS 给非生产包加水印防混包）。

### 2. i18n 国际化：ARB + gen-l10n ⭐⭐⭐

iOS 对照：`.arb` ≈ **Localizable.strings / String Catalog**；gen-l10n 生成的
`AppLocalizations` ≈ Xcode 从字符串目录生成的本地化访问代码；但这里**类型安全**——
key 拼错**编译期就报错**，不像 `NSLocalizedString` 拼错了静默返回原字符串。

装配三件事：

```yaml
# pubspec.yaml
dependencies:
  flutter_localizations: { sdk: flutter }
  intl: any
flutter:
  generate: true            # 开启 gen-l10n 代码生成
```

```yaml
# l10n.yaml —— 告诉 gen-l10n 从哪读、生成到哪、类叫什么
arb-dir: lib/l10n
template-arb-file: app_en.arb
output-localization-file: app_localizations.dart
output-class: AppLocalizations
nullable-getter: false      # of(context) 返回非空，少写一堆 !
```

```dart
// app.dart：把 delegate + 支持语言接进 MaterialApp
localizationsDelegates: AppLocalizations.localizationsDelegates,
supportedLocales: AppLocalizations.supportedLocales,

// 用的时候：
final l10n = AppLocalizations.of(context);
Text(l10n.navHome);                    // 简单文案
Text(l10n.currentEnv('dev'));          // 占位符：{name} → String(format:)
Text(l10n.cartItemCount(count));       // ICU 复数：0/1/多 自动选措辞
```

- **ICU 复数**是重点：一个 key 里用 `{count, plural, =0{...} =1{...} other{...}}`
  就搞定单复数/零态。≈ iOS 的 `.stringsdict`，但写在**同一个 arb 键**里，不用单独文件。
  英文 `=1{item}` vs `other{items}`、中文只需 `=0` 和 `other`（中文没有单复数变形）。
- 生成物 `lib/l10n/app_localizations*.dart` 是 gen-l10n 产出的，**和 `.g.dart` 一样提交进 git**
  （本项目惯例：代码生成物入库，免得别人 clone 后没跑生成就编译不过）。

### 3. 测试环境的语言切换 ⭐

Tab 标签一旦国际化，**widget 测试就受默认语言影响**：flutter test 默认 `en`，
原来断言中文 `'首页'` 的测试会挂。正确姿势是**在测试里强制设备语言**：

```dart
tester.platformDispatcher.localesTestValue = const [Locale('zh')];
addTearDown(tester.platformDispatcher.clearLocalesTestValue);
```

- 坑：要用 **`localesTestValue`（复数）** 而不是 `localeTestValue`（单数）——
  MaterialApp 的 locale 解析读的是 `platformDispatcher.locales`（列表）。单数那个设了不生效。
- iOS 对照：≈ XCUITest 用 `launchArguments = ["-AppleLanguages", "(zh-Hans)"]` 测指定语言界面。

## 二、新 API / 概念速查表

| API / 概念 | iOS 类比 | 怎么用 | 坑 |
|---|---|---|---|
| `String.fromEnvironment('K', defaultValue: …)` | `#if DEBUG` / xcconfig 变量 | 读 `--dart-define=K=v` 的编译期常量 | 只认**编译期**；运行时永远读不到、也改不了；必须 const 上下文 |
| `--dart-define=FLAVOR=prod` | Scheme / Build Config | run/build 时传，选环境 | 不传就用 defaultValue；打包别忘了带 |
| `MaterialApp.builder` | 给 window 套一层根容器 | `(context, child) => 包一层(child)` | `child` 可空，返回前记得 `child!`；别在这里做路由 |
| `Banner` widget | 非生产包的水印角标 | `Banner(message, location, color, child)` | 和 `debugShowCheckedModeBanner` 是两码事，后者是右上角 DEBUG 丝带 |
| `AppLocalizations.of(context)` | `NSLocalizedString` | 取当前语言文案 | 拼错 key 编译期报错（好事）；`nullable-getter:false` 后不用 `!` |
| ARB `{n, plural, ...}` | `.stringsdict` | 一个 key 表达单复数/零态 | 中文只要 `=0`/`other`；英文补 `=1`；占位符要在 `@key` 里声明 type |
| `generate: true` + `l10n.yaml` | String Catalog 自动生成 | pub get / build 时自动跑 gen-l10n | 改了 arb 要重新 `flutter gen-l10n` 或 pub get 才生效 |
| `platformDispatcher.localesTestValue` | XCUITest `-AppleLanguages` | 测试里强制语言 | 用复数版；记得 tearDown 清掉 |

2. 给 staging 环境的 `baseUrl` 改成一个假的预发域名，用 `flutter run --dart-define=FLAVOR=staging`
   跑，观察右上角角标从 DEV 变 STAGING、以及网络请求打到了不同域名（会失败，但日志里能看到 URL 变了）。

   一个键一个键搬，不必一次性全改完。
