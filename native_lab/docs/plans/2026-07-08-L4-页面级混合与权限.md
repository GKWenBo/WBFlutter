# NativeLab L4：页面级混合 + 权限 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **教学项目特别说明**：本计划的"执行"是一堂课——边写边讲（iOS 类比先行）。真机验证需学员在模拟器上操作（本课点"扫码"→ 授权 → 在原生扫码页选一个码 → 看结果回到 Flutter）。

**Goal:** 从前三课的"数据过桥"跨到 **页面级混合**——Flutter 让原生**接管整块屏幕**（`present` 一个原生 `UIViewController`），用户在原生页面里操作完，把结果**回传**给 Flutter。同时第一次碰**运行时权限**：扫码要相机权限，走完"申请 → 授权/拒绝 → 拒绝后降级"的真实流程。落地场景是电商最常见的**扫码**（扫商品条码/优惠券）。

**场景（企业真实）：** Flutter 商品页点"扫码"，唤起原生扫码页（真机上是 `AVCaptureSession` 相机预览 + 二维码识别），扫到码后回传给 Flutter 去查商品。本课在**模拟器**上跑，没有相机硬件，所以扫码页用**模拟页**（预置按钮 + 输入框代替"扫到"），但**权限申请、present/dismiss、结果回传**这条主线全部真实——这三样才是本课要学的东西，相机采集留到真机。

**Architecture:** 沿用分层。Dart 侧 `ScanBridge`（包第四条 `MethodChannel`，`scan()` 返回强类型 `ScanOutcome`）+ L4 页面（`StatefulWidget` + `setState`，够用不上 Riverpod）；iOS 侧 `ScanBridge`（MethodChannel handler：查/求相机权限 → present `MockScannerViewController` → 把 `FlutterResult` **暂存**，等用户扫完/取消再回）；Android 侧 `ActivityResultLauncher` + 运行时权限对照。**现在 App 挂四条 channel（device_info / analytics / network_status / scanner），注册并列追加。**

**★ 与前三课的关键差异（本课核心心智）：**
- L1/L2 的 `result(...)` 在 handler 里**立刻**就回；本课的 `result` 是**延迟**的——present 出去后，要把它**暂存成 `pendingResult`**，等用户在原生页操作完（回调里）才调用。`FlutterResult` **只能调用一次**，所以要单次在飞行守卫、回完置空。
- L1–L3 是"数据/流过桥"，本课是"**整个原生页面接管屏幕**"。这跟 L6 的 `PlatformView`（把原生 view 嵌进 Flutter widget 树里）是两条不同的路子——**页面级 vs 视图级**，L6 会正式对比。

**Tech Stack:** `MethodChannel.invokeMethod` + 暂存 `FlutterResult` 异步回调；iOS `AVFoundation`(`AVCaptureDevice.authorizationStatus`/`requestAccess`)、`UIViewController` present/dismiss、`Info.plist` 的 `NSCameraUsageDescription`；Android `ActivityResultLauncher` + `ActivityResultContracts.RequestPermission`。测试**回到 MethodChannel 的 `setMockMethodCallHandler`**（对比 L3 的 `setMockStreamHandler`）——页面级混合本质仍是"一次请求一个结果"，只是结果来得晚。

```dart
/// 扫码结果的强类型收口。原生把三种情形分别用"回字符串 / 回 null / 抛
/// PERMISSION_DENIED"表达，桥接层统一收成 sealed 类，UI 用 switch 穷尽处理
/// （延续 L1/L2/L3 的"裸值 → 强类型"收口思路；sealed 让 switch 漏分支时编译报错）。
sealed class ScanOutcome {
  const ScanOutcome();
}

/// 扫到码。
class ScanSuccess extends ScanOutcome {
  const ScanSuccess(this.code);
  final String code;
}

/// 用户主动取消（原生返回 null）。
class ScanCancelled extends ScanOutcome {
  const ScanCancelled();
}

/// 相机权限被拒（原生返回 PERMISSION_DENIED 错误）。
class ScanPermissionDenied extends ScanOutcome {
  const ScanPermissionDenied();
}
```

`lib/lessons/l4/scan_bridge.dart`：

```dart
import 'package:flutter/foundation.dart';
import 'package:flutter/services.dart';

import 'scan_outcome.dart';

/// L4 扫码桥（Dart 侧）。第四条 channel。
/// 页面级混合：原生不是回一个数据，而是"接管整块屏幕"present 一个原生扫码页，
/// 等用户扫完/取消后再回传结果——但 Dart 侧用法仍是一次 invokeMethod。
/// 关键差异（对比 L1/L2）：原生那次 result 是"延迟"的（用户操作后才回来）。
class ScanBridge {
  ScanBridge._();

  @visibleForTesting
  static const MethodChannel channel =
      MethodChannel('com.wenbo.native_lab/scanner');

  /// 打开原生扫码页，返回强类型结果。
  /// 约定：扫到码 → 回 code 字符串；取消 → 回 null；权限被拒 → 抛 PERMISSION_DENIED。
  static Future<ScanOutcome> scan() async {
    try {
      final code = await channel.invokeMethod<String>('scan');
      return code == null ? const ScanCancelled() : ScanSuccess(code);
    } on PlatformException catch (e) {
      if (e.code == 'PERMISSION_DENIED') {
        return const ScanPermissionDenied();
      }
      rethrow; // 其它错误（如 ALREADY_SCANNING）不吞，交上层
    }
  }
}
```

```swift
import AVFoundation

/// L4 扫码桥（原生侧）。页面级混合：present 一个原生 VC，把 FlutterResult
/// 暂存，等用户操作完再回。result 只能调一次，所以要单次在飞行 + 回完置空。
enum ScanBridge {
  private static var pendingResult: FlutterResult?

  static func register(messenger: FlutterBinaryMessenger) {
    let channel = FlutterMethodChannel(
      name: "com.wenbo.native_lab/scanner", binaryMessenger: messenger)
    channel.setMethodCallHandler { call, result in
      switch call.method {
      case "scan": handleScan(result: result)
      default: result(FlutterMethodNotImplemented)
      }
    }
  }

  private static func handleScan(result: @escaping FlutterResult) {
    // 上一次还没回就直接拒，避免 pendingResult 被覆盖（result 只能调一次）。
    if pendingResult != nil {
      result(FlutterError(code: "ALREADY_SCANNING", message: "已在扫码中", details: nil))
      return
    }
    // 相机权限：真机扫码要开 AVCaptureSession，先拿权限。本课扫码页是模拟的，
    // 但权限这条线走真实流程（这就是本课的"权限"知识点）。
    switch AVCaptureDevice.authorizationStatus(for: .video) {
    case .authorized:
      presentScanner(result: result)
    case .notDetermined:
      AVCaptureDevice.requestAccess(for: .video) { granted in
        DispatchQueue.main.async {   // 回调在任意线程，present 必须回主线程
          granted ? presentScanner(result: result)
                  : result(FlutterError(code: "PERMISSION_DENIED", message: "相机权限被拒", details: nil))
        }
      }
    default: // .denied / .restricted
      result(FlutterError(code: "PERMISSION_DENIED", message: "相机权限被拒", details: nil))
    }
  }

  private static func presentScanner(result: @escaping FlutterResult) {
    guard let top = topViewController() else {
      result(FlutterError(code: "NO_UI", message: "找不到可 present 的控制器", details: nil))
      return
    }
    pendingResult = result
    let scanner = MockScannerViewController { code in finish(with: code) }
    let nav = UINavigationController(rootViewController: scanner)
    nav.modalPresentationStyle = .fullScreen
    top.present(nav, animated: true)
  }

  // 回传出口：扫到 code(String) → Dart 得字符串；取消(nil) → Dart 得 null。
  private static func finish(with code: String?) {
    pendingResult?(code)
    pendingResult = nil
  }

  // Flutter + Scene 世界里"从哪 present"：取前台 windowScene 的 rootVC 再顺到最顶层。
  private static func topViewController() -> UIViewController? {
    let scene = UIApplication.shared.connectedScenes
      .first { $0.activationState == .foregroundActive } as? UIWindowScene
    var top = scene?.keyWindow?.rootViewController
    while let presented = top?.presentedViewController { top = presented }
    return top
  }
}

/// 模拟扫码页。真机上这里是 AVCaptureSession 预览 + AVCaptureMetadataOutput
/// 识别二维码；模拟器没相机，用预置按钮 + 输入框替代"扫到"，但 present/dismiss/
/// 回传这条页面级混合主线完全真实。dismiss 完成回调里才调 onFinish（先收页面再回值）。
final class MockScannerViewController: UIViewController {
  private let onFinish: (String?) -> Void
  private let field = UITextField()

  init(onFinish: @escaping (String?) -> Void) {
    self.onFinish = onFinish
    super.init(nibName: nil, bundle: nil)
  }
  required init?(coder: NSCoder) { fatalError("init(coder:) 未使用") }

  override func viewDidLoad() {
    super.viewDidLoad()
    view.backgroundColor = .systemBackground
    title = "模拟扫码（本机无相机）"
    navigationItem.leftBarButtonItem = UIBarButtonItem(
      barButtonSystemItem: .cancel, target: self, action: #selector(cancelTapped))

    field.placeholder = "手动输入一个码"
    field.borderStyle = .roundedRect
    let preset1 = makeButton("扫到 SKU-10086") { [weak self] in self?.scanned("SKU-10086") }
    let preset2 = makeButton("扫到 COUPON-8" ) { [weak self] in self?.scanned("COUPON-8") }
    let confirm = makeButton("确认输入的码") { [weak self] in
      self?.scanned(self?.field.text?.isEmpty == false ? self!.field.text! : "EMPTY")
    }
    let stack = UIStackView(arrangedSubviews: [field, preset1, preset2, confirm])
    stack.axis = .vertical; stack.spacing = 16; stack.translatesAutoresizingMaskIntoConstraints = false
    view.addSubview(stack)
    NSLayoutConstraint.activate([
      stack.centerYAnchor.constraint(equalTo: view.centerYAnchor),
      stack.leadingAnchor.constraint(equalTo: view.leadingAnchor, constant: 32),
      stack.trailingAnchor.constraint(equalTo: view.trailingAnchor, constant: -32),
    ])
  }

  @objc private func cancelTapped() {
    dismiss(animated: true) { [onFinish] in onFinish(nil) }       // 取消 = nil
  }
  private func scanned(_ code: String) {
    dismiss(animated: true) { [onFinish] in onFinish(code) }      // 扫到 = code
  }
  private func makeButton(_ title: String, _ action: @escaping () -> Void) -> UIButton {
    let b = UIButton(type: .system)
    b.setTitle(title, for: .normal)
    b.titleLabel?.font = .systemFont(ofSize: 18, weight: .medium)
    b.addAction(UIAction { _ in action() }, for: .touchUpInside)
    return b
  }
}
```

- [ ] **Step 1: 追加 scanner MethodChannel + launcher 模式**（configureFlutterEngine 内 + Activity 字段）

Android 的两处不同要点在注释里讲透：
```kotlin
// 关键差异 1：原生页面用 startActivity/ActivityResultLauncher 打开另一个 Activity,
//   不是 iOS 那样 present VC；结果通过 onActivityResult/launcher 回调回来。
// 关键差异 2：launcher 必须在 STARTED 之前注册（这里在 configureFlutterEngine/onCreate 期），
//   所以 pendingResult 也要暂存成字段，等 launcher 回调再 result.success(...)。
private var pendingResult: MethodChannel.Result? = null
private lateinit var scanLauncher: ActivityResultLauncher<Intent>
private lateinit var cameraPermLauncher: ActivityResultLauncher<String>
```
在 `configureFlutterEngine` 里 `registerForActivityResult` 注册两个 launcher（扫码 Activity 的返回、相机权限的授予），MethodChannel `scan` 里：查 `ContextCompat.checkSelfPermission` → 有权限就 `scanLauncher.launch(Intent(this, ScannerActivity::class.java))`、没有就 `cameraPermLauncher.launch(Manifest.permission.CAMERA)`。扫码 Activity 回来 `RESULT_OK` 取 `data.getStringExtra("code")` → `pendingResult?.success(code)`；取消 → `success(null)`；权限拒 → `error("PERMISSION_DENIED", ...)`。

- [ ] **Step 2: 讲解双端差异**（写进课时文档"双端对照"）：
  ① 打开原生页：iOS `present(VC)` vs Android `startActivity`/`ActivityResultLauncher`；② 回传：iOS 暂存 `FlutterResult` 在 dismiss 回调里回 vs Android 暂存 `Result` 在 `onActivityResult`/launcher 回调里回（两端都因"结果延迟到达"要暂存）；③ 权限：iOS `AVCaptureDevice.requestAccess` + `NSCameraUsageDescription` vs Android `ActivityResultContracts.RequestPermission` + Manifest `<uses-permission CAMERA>`；④ present 位置：iOS 要找 topVC，Android launcher 必须在 STARTED 前注册。
