> **教学项目特别说明**：本计划的"执行"是一堂课——边写边讲（iOS 类比先行），验证截图给学员看。真机效果验证需要学员在模拟器上亲手点按钮，届时截图确认。

**Goal:** 吃透 MethodChannel 底层的 **StandardMessageCodec** 类型系统：Dart↔Swift↔Kotlin 三端类型映射表；把**复杂嵌套结构**（事件名 + 属性 Map，属性里混 String/int/double/bool/List/嵌套 Map）无损传给原生；用 **Uint8List** 做二进制零拷贝传输；亲手复现并讲透"**类型不匹配崩溃**"（int/double 混用、Android 的 Int/Long 边界）这一企业高频事故。

**场景（企业真实）：** 埋点桥——Flutter 的埋点统一走原生已接好的统计 SDK（友盟/GA/自研）。Flutter 侧构造事件，通过 channel 灌给原生"SDK"（本课用内存 buffer 模拟），原生回自增序号；Dart 侧还能把 buffer 里的事件全拉回来，验证往返编解码无损。

**Architecture:** 沿用 L1 的分层。Dart 侧 `AnalyticsBridge`（静态方法包一条新 channel `com.wenbo.native_lab/analytics`）+ `AnalyticsEvent` 模型 + L2 页面；iOS 侧新增 `AnalyticsBridge` enum（内存 buffer 模拟 SDK），与 L1 的 `DeviceInfoBridge` 并列注册在 AppDelegate.swift；Android 侧 MainActivity 追加第二条 channel 对照实现。**关键：一个 App 现在挂两条 channel，注册代码要并列，别互相覆盖。**

**Tech Stack:** MethodChannel + StandardMessageCodec（默认 codec，本课主角）；`invokeMethod` 第二参传复杂参数；`invokeListMethod`/`invokeMapMethod` 做返回值收窄；`Uint8List` ↔ `FlutterStandardTypedData` ↔ `ByteArray`。测试用 `TestDefaultBinaryMessengerBinding` mock。

`lib/lessons/l2/analytics_event.dart`：

```dart
/// 埋点事件模型。企业里事件 = 事件名 + 一坨属性；属性值类型五花八门
/// （字符串/数字/布尔/数组/嵌套对象），这正好把 codec 的类型系统全踩一遍。
class AnalyticsEvent {
  const AnalyticsEvent({required this.name, this.properties = const {}});

  final String name;

  /// 属性值只能是 StandardMessageCodec 支持的类型（见课时文档映射表），
  /// 否则编码期就会抛。故意用 Object? 而不是 dynamic：逼调用方心里有数。
  final Map<String, Object?> properties;

  /// 过桥前拍平成 Map——channel 只认 codec 支持的裸结构，不认自定义类。
  Map<String, Object?> toMap() => {'name': name, 'properties': properties};

  /// 原生传回来的是 Map<Object?, Object?>，先 cast 收窄再收口成模型。
  factory AnalyticsEvent.fromMap(Map<String, Object?> map) => AnalyticsEvent(
        name: map['name'] as String? ?? '',
        properties:
            (map['properties'] as Map?)?.cast<String, Object?>() ?? const {},
      );
}
```

`lib/lessons/l2/analytics_bridge.dart`：

```dart
import 'dart:typed_data';

import 'package:flutter/foundation.dart';
import 'package:flutter/services.dart';

import 'analytics_event.dart';

/// L2 埋点桥（Dart 侧）。与 L1 的 DeviceInfoBridge 同构，但这次
/// 参数/返回都是复杂结构，重点在 codec 怎么搬运它们。
class AnalyticsBridge {
  AnalyticsBridge._();

  @visibleForTesting
  static const MethodChannel channel =
      MethodChannel('com.wenbo.native_lab/analytics');

  /// 上报一个事件。invokeMethod 第二参 = 传给原生的参数，
  /// 这里传一个嵌套 Map，codec 会递归编码每一层。原生回自增序号。
  static Future<int> logEvent(AnalyticsEvent event) async {
    final seq = await channel.invokeMethod<int>('logEvent', event.toMap());
    return seq ?? -1;
  }

  /// 把原生缓存的所有事件拉回来。invokeListMethod 帮我们把
  /// List<Object?> 收窄；每个元素仍是 Map<Object?,Object?>，再各自 fromMap。
  static Future<List<AnalyticsEvent>> fetchLoggedEvents() async {
    final list = await channel.invokeListMethod<Object?>('fetchLoggedEvents');
    if (list == null) return const [];
    return list
        .map((e) => AnalyticsEvent.fromMap(
            (e as Map).cast<String, Object?>()))
        .toList();
  }

  /// 上传一段二进制日志。Uint8List 走 codec 的 typed-data 分支，
  /// 零拷贝、不 base64——传字节流（缩略图/protobuf）就该用它。
  static Future<int> uploadRawLog(Uint8List bytes) async {
    final n = await channel.invokeMethod<int>('uploadRawLog', bytes);
    return n ?? 0;
  }
}
```

```swift
/// L2 埋点桥（原生侧）。内存 buffer 模拟一个统计 SDK。
enum AnalyticsBridge {
  // 模拟统计 SDK 的事件缓存。真实场景这里换成 SDK 的 track 调用。
  private static var buffer: [[String: Any]] = []

  static func register(messenger: FlutterBinaryMessenger) {
    let channel = FlutterMethodChannel(
      name: "com.wenbo.native_lab/analytics", // 与 Dart 逐字符一致
      binaryMessenger: messenger)
    channel.setMethodCallHandler(handle)
  }

  private static func handle(_ call: FlutterMethodCall, result: @escaping FlutterResult) {
    switch call.method {
    case "logEvent":
      // Dart 的 Map<String,Object?> 经 codec 解成 NSDictionary → [String: Any]。
      // 数字统一是 NSNumber：Int/Double 取值时用 .intValue/.doubleValue。
      guard let map = call.arguments as? [String: Any] else {
        result(FlutterError(code: "BAD_ARGS", message: "logEvent 期望 Map", details: nil))
        return
      }
      buffer.append(map)
      result(buffer.count) // 自增序号；Int 会被编码回 Dart 的 int
    case "fetchLoggedEvents":
      result(buffer) // [[String:Any]] → Dart List<Map>
    case "uploadRawLog":
      // 二进制：Dart Uint8List → FlutterStandardTypedData，.data 是 Data。
      guard let typed = call.arguments as? FlutterStandardTypedData else {
        result(FlutterError(code: "BAD_ARGS", message: "uploadRawLog 期望二进制", details: nil))
        return
      }
      result(typed.data.count)
    default:
      result(FlutterMethodNotImplemented)
    }
  }
}
```

- [ ] **Step 2: 讲解双端差异**（写进课时文档"双端对照"）
① 数字桥接：iOS 全是 NSNumber，Kotlin 分 Integer/Long/Double——**小整数是 Integer、超 int32 的是 Long**，`as Int` 对大数会 ClassCastException，稳妥 `(x as Number).toLong()`；② 二进制：iOS 是 `FlutterStandardTypedData.data`，Kotlin 直接是 `ByteArray`；③ Map：iOS `[String: Any]`（NSDictionary），Kotlin `Map<String, Any?>`（HashMap）。

- [ ] **Step 1: 写课时文档**（六段模板，L2 核心 = 类型映射表）

1. **本课要掌握什么**：StandardMessageCodec 是 MethodChannel 默认的"搬运工"，只认固定的一张类型表；复杂参数怎么传（第二参 + 嵌套）；二进制为什么用 Uint8List（零拷贝，不 base64）；三大类型事故（int/double 混用、Android Int/Long 边界、Map<Object?,Object?> 直接当强类型 Map 用）。
2. **★ 类型映射大表**（本课核心，必须精确）：

   | Dart | Swift (iOS) | Kotlin (Android) | 备注 |
   |---|---|---|---|
   | null | NSNull / nil | null | |
   | bool | NSNumber → Bool | Boolean | |
   | int (≤32bit) | NSNumber → Int | Integer | Kotlin 小整数是 Int |
   | int (>32bit) | NSNumber → Int | Long | **Kotlin 大整数是 Long，坑** |
   | double | NSNumber → Double | Double | 与 int 不可混 |
   | String | String (NSString) | String | |
   | Uint8List | FlutterStandardTypedData | ByteArray | 二进制零拷贝 |
   | Int32List/Int64List/Float64List | FlutterStandardTypedData | IntArray/LongArray/DoubleArray | 数值数组高效传 |
   | List\<Object?\> | NSArray / [Any] | List\<Any?\> | |
   | Map\<Object?,Object?\> | NSDictionary / [AnyHashable:Any] | Map\<Any?,Any?\> | 回 Dart 是 Object? 键值 |

3. **控件/API 速查表**：`invokeMethod(m, args)` 第二参、`invokeListMethod`/`invokeMapMethod` 收窄、`Uint8List`/`FlutterStandardTypedData`/`ByteArray`、`.cast<K,V>()`；各配 iOS 类比 + 坑。

6. **自测清单**：① Dart 的 `int` 传到 Swift/Kotlin 各变成什么？为什么 Android 侧 `as Int` 有时会崩？② 原生返回 double 而 Dart 用 `invokeMethod<int>` 接，会发生什么、在哪一端崩？③ 传一张缩略图字节该用什么类型，走 base64 String 有何不同？④ `fetchLoggedEvents` 返回的 List 里每个元素的静态类型是什么，为什么要 `.cast`？
