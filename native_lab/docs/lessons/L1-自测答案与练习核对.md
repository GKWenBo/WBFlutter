## 一、自测清单答案

### Q1. channel 名出现在项目哪几处？错一个字符会怎样，Dart 侧报哪种异常？

**出现三处，一端一处，必须逐字符一致：**

| 端 | 位置 | 代码 |
|---|---|---|
| Dart | [device_info_bridge.dart:17](../../app/lib/lessons/l1/device_info_bridge.dart) | `MethodChannel('com.wenbo.native_lab/device_info')` |
| iOS | [AppDelegate.swift:28](../../app/ios/Runner/AppDelegate.swift) | `FlutterMethodChannel(name: "com.wenbo.native_lab/device_info", ...)` |
| Android | [MainActivity.kt:12](../../app/android/app/src/main/kotlin/com/wenbo/native_lab/MainActivity.kt) | `MethodChannel(..., "com.wenbo.native_lab/device_info")` |

（测试里用的是 `DeviceInfoBridge.channel` 这个引用，不是字面量，所以不算第四处——这正是把 channel 名收成一个 `const` 常量的价值：Dart 侧只有一处真源。）

**错一个字符 → Dart 侧收到 `MissingPluginException`。** 因为频道名对不上，等于原生侧**没有任何 handler 认领这条频道**，messenger 找不到接收方。

⚠️ 关键区分——`MissingPluginException` 其实覆盖**两种**情况：
1. **频道名**错了 / 原生压根没注册 → 没人认领这条 channel；
2. 频道名对、但**方法名**错了 → 原生走到 `default`/`else` 返回 `FlutterMethodNotImplemented`/`notImplemented()`，Dart 侧同样表现为 `MissingPluginException`。

对照：**业务失败**（如模拟器没电池，原生主动 `result(FlutterError)`）才是 `PlatformException`。一句话记忆：**"桥没通" = MissingPlugin，"桥通了但业务说不行" = PlatformException**。

### Q2. 原生 `result(FlutterError(...))` 到 Dart 侧变成什么类型？捕获它的 catch 怎么写？

变成 **`PlatformException`**，字段一一对应：`FlutterError(code:message:details:)` → `PlatformException(code, message, details)`。

调用方的兜底写法（`on X catch` 只捕 `PlatformException`，别用宽 catch 吞掉 `MissingPluginException` 那类"桥没通"的错）：

```dart
try {
  final level = await DeviceInfoBridge.fetchBatteryLevel();
  // 用 level...
} on PlatformException catch (e) {
  // e.code == 'UNAVAILABLE'，e.message == '拿不到电量（模拟器没有电池）'
  if (e.code == 'UNAVAILABLE') showBatteryUnavailable();
}
```

测试里就是这么断言的（[l1_device_info_test.dart:53](../../app/test/l1_device_info_test.dart)）：
`isA<PlatformException>().having((e) => e.code, 'code', 'UNAVAILABLE')`。

### Q3. 原生 handler 跑在哪条线程？里面做耗时磁盘 IO 会怎样、该怎么办？

**跑在 Platform 线程 == iOS 主线程**（Android 侧同理 = 主线程）。见 [AppDelegate.swift:34](../../app/ios/Runner/AppDelegate.swift) 注释。

**直接在闭包里做耗时磁盘 IO 的后果：** 阻塞主线程 → UI 卡死/掉帧，还会堵住这条 channel 上后续所有消息；iOS 上卡太久甚至会被 watchdog 杀进程。

**正确做法——把重活挪出 Platform 线程，做完再回主线程调 result：**

```swift
case "readBigFile":
  DispatchQueue.global().async {          // 后台线程做磁盘 IO
    let data = heavyDiskIO()
    DispatchQueue.main.async { result(data) }  // 回主线程交付结果
  }
```

Android 对应：丢到后台线程 / 协程，回 `runOnUiThread { result.success(...) }`。

补充两点：① channel **本身没有超时**，原生迟迟不 result，Dart 的 Future 就永久挂着，需要时 Dart 侧自己包 `.timeout(Duration(...))`；② 每条路径 `result` 必须**调且只调一次**，漏调=Future 永挂，多调=运行时报错。

### Q4. 为什么 3 个桥单测不起模拟器也能跑通？mock 挂在哪一层？

因为 `setMockMethodCallHandler` **在 BinaryMessenger 这一层把出站消息拦截了**，消息根本没机会跨到原生侧。挂载点是：

```dart
TestDefaultBinaryMessengerBinding.instance.defaultBinaryMessenger
    .setMockMethodCallHandler(DeviceInfoBridge.channel, handler);
```

见 [l1_device_info_test.dart:14](../../app/test/l1_device_info_test.dart) 的 `mockNative`。

**数据流：** `invokeMethod` → codec 编码 → 交给（测试用的）binary messenger → messenger 发现这条 channel 被 mock 了，**不转发给平台**，直接调你的 mock handler → mock 的返回值经 codec 解码 → `invokeMethod` 的 Future 完成。

于是 **Dart 侧的所有逻辑都真跑了**（编码、解码、`DeviceInfo.fromMap` 收口、`PlatformException` 转换），唯独没有原生运行时、没有模拟器——所以一行 Swift 都没编译，CI 上零模拟器就能测。

**记忆点：mock 挂在 BinaryMessenger 上，不是挂在 MethodChannel 对象上**——channel 只是那把"路由钥匙"（`setMockMethodCallHandler` 的第一个参数），真正被替换掉转发行为的是 messenger。

> 备注：本课初版是 3 桥单测 + 2 widget test；你做完课后练习补了 1 条 uptime 单测，现共 **4 桥单测 + 2 widget test = 6 条，全绿**。

**两条点评（都是加分项 / 真实工程经验）：**

1. **命名你用了 `getSystemUpTime`（大写 T），跟文档示例的 `getSystemUptime` 不同——但你四端全都统一成了同一个串，所以照样跑通。** 这恰好亲手验证了 Q1 的核心：MethodChannel 认的是"逐字符一致"，方法名叫什么不重要，四端一致才重要。假如你当时只改了 Dart 没改原生（或大小写各写各的），就会在真机上撞 `MissingPluginException`——这就是手写 channel 的"魔法字符串"漂移痛点，也是 L5 用 Pigeon 代码生成来根除的动机。

2. **修掉一个夹带问题：** `device_info_bridge.dart` 第 1 行被 IDE 自动补了个 `import 'dart:ffi';`（你敲 `double` 时误触发的自动导入，本文件的 `double` 是 Dart 核心类型，与 ffi 无关）。已删除，analyze 恢复 0 issue。**经验：Dart 的 `double`/`int`/`num` 都是核心类型，永远不需要 import；VS Code 弹出的 `dart:ffi` 自动导入几乎总是误报，随手删。**

**唯一没覆盖到的（非必须，指出即可）：** 你补了 Dart 侧 mock 单测，但没在页面上加"获取开机时长"按钮做真机验证。练习本身只要求到 mock 测试，所以判过；真要端到端验证，可仿照现有两个按钮加第三个（留作 L2 手感练习）。
