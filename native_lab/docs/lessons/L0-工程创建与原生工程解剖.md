# L0 工程创建与原生工程解剖

> 企业场景：入职第一天，接手一个 Flutter 工程，你得先看懂 `ios/` 和 `android/` 到底是什么、Dart 代码是怎么跑进手机里的——这决定了你后面九课的所有原生交互代码往哪写。

## 一、本课要掌握什么

**1. `ios/` 就是一个完整的 Xcode 工程，`android/` 就是一个完整的 Gradle 工程。**

`flutter create` 没有魔法：它生成的 `ios/Runner.xcworkspace` 你可以直接用 Xcode 打开、编译、断点调试，和你写了多年的原生工程没有本质区别。`flutter run` 只是对 `xcodebuild` / `gradle` 的包装，外加把 Dart 产物塞进构建流程。

**2. Dart 代码以什么形式进到 iOS 包里？**

- Debug 模式：Dart 代码由 JIT 运行（所以才有热重载）；
- Release 模式：Dart 被 AOT 编译成 arm64 机器码，打成 **`App.framework`** 塞进 .app 包，旁边还有引擎本体 **`Flutter.framework`**。
- 把 Dart 产物接进 Xcode 构建的桥梁是 `ios/Flutter/Generated.xcconfig`（flutter 工具生成，记录 SDK 路径和构建参数），所以先跑过一次 `flutter run` / `flutter build`，Xcode 才能独立构建成功。

**3. FlutterEngine 与 FlutterViewController 的关系（后面每一课的地基）。**

- **FlutterEngine**：Dart 的运行时，负责跑 Dart 代码 + 渲染。
- **FlutterViewController**：一个普通的 `UIViewController` 子类，把引擎渲染的内容显示出来。App 的 `rootViewController` 就是它。
- 一句话心智模型：**引擎是播放器内核，FlutterViewController 是播放窗口**。L8/L9 的 add-to-app 玩的就是"自己管理引擎和窗口"。

**4. 三条关键线程（L1 讲回调线程的伏笔）。**

| 线程 | 跑什么 | iOS 类比 |
|---|---|---|
| Platform 线程 | 原生代码、channel 的原生侧回调 | **就是 iOS 主线程** |
| UI 线程 | 你的全部 Dart 代码 | 类似一个专用的串行队列，但跑的是另一门语言的运行时 |
| Raster 线程 | 把图层栅格化上屏 | 类似 Core Animation 的 render server 干的活 |

关键认知：**Dart 代码不在 iOS 主线程上跑**。所以 channel 通信本质是跨线程消息传递——这就是 L1 的入口。

**5. 新模板的两个新鲜细节（和老教程不一样的地方）。**

- iOS 侧已经有 `SceneDelegate.swift`，接入了 UIScene 生命周期（和你现在的原生工程一致）。
- `AppDelegate` 除了继承 `FlutterAppDelegate`，还实现了 `FlutterImplicitEngineDelegate`——插件注册挪到了 `didInitializeImplicitFlutterEngine` 回调里（老模板是在 `didFinishLaunching` 里直接 `GeneratedPluginRegistrant.register(with: self)`）。"隐式引擎"指的是模板帮你自动创建的那台引擎，与之相对的就是 L8/L9 里我们自己显式创建的引擎。

## 二、控件 / API 速查表

### Flutter 侧（本课新出现）

| 控件/API | iOS 类比 | 用法 & 关键参数 | 易踩的坑 |
|---|---|---|---|
| `MaterialApp` | `UIApplication` + `UIWindow` + 全局主题的合体 | `title`、`theme: ThemeData(colorSchemeSeed:)`、`home` | `title` 不是界面标题，是任务切换器里的名字；界面标题在 `AppBar` |
| `Scaffold` | 带导航栏的 VC 模板（`UIViewController` + `UINavigationBar` 布局骨架） | `appBar`、`body`、`floatingActionButton` | 没有 Scaffold 的页面弹 SnackBar 会崩；每个"页面"基本都该有一个 |
| `ListView.separated` | `UITableView` + 自动分隔线 | `itemCount`、`itemBuilder`、`separatorBuilder` | 懒加载：屏幕外的 item **不会 build**（写测试时找不到屏幕外的文本就是这个原因） |
| `ListTile` | `UITableViewCell` 的 `.subtitle` 样式 | `leading`/`title`/`subtitle`/`trailing`/`onTap` | 想要"禁用但可点提示"不能用 `enabled: false`（它会吞掉 onTap） |
| `CircleAvatar` | 圆形头像 `UIImageView`（layer.cornerRadius 那套的封装） | `child` 或 `backgroundImage` | 放文字时字号跟随主题，不跟 radius 自动缩放 |
| `SnackBar` + `ScaffoldMessenger` | 底部 toast/HUD；Messenger 类比全局单例管理器 | `ScaffoldMessenger.of(context).showSnackBar(...)` | 连点会排队，先 `hideCurrentSnackBar()` 再 show 体验更好 |
| `Navigator` + `MaterialPageRoute` | `UINavigationController.pushViewController` | `Navigator.of(context).push(MaterialPageRoute(builder:))` | Flutter 页面没有 VC 这层，`builder` 返回的就是页面 Widget 本身 |
| `WidgetBuilder` | `(BuildContext) -> Widget` 的 typedef，类比 `() -> UIViewController` 闭包 | 存起来延迟构建页面 | 它是函数不是 Widget，registry 里存它才能"用时才建页面" |
| switch 表达式 | Swift 的 `switch` 表达式（5.9 的 if/switch expression） | `trailing: switch (status) { A => x, B => y }` | 必须穷举；这是 Dart 3 语法，老教程里看不到 |

### 原生侧（本课新出现的概念）

| 概念/文件 | iOS 类比 | 说明 | 易踩的坑 |
|---|---|---|---|
| `Runner.xcworkspace` | 就是你熟悉的 workspace | 含 CocoaPods 时的打开入口 | 打开 `.xcodeproj` 会缺 Pods 依赖，编不过 |
| `FlutterAppDelegate` | `UIApplicationDelegate` 的 Flutter 版基类 | 帮你处理引擎、插件生命周期分发 | 自己的 AppDelegate 逻辑记得调 `super` |
| `FlutterViewController` | 普通 `UIViewController` 子类 | 引擎内容的"播放窗口" | 它不画 UIKit 控件，里面是引擎自绘的一整块 surface |
| `GeneratedPluginRegistrant` | 没有直接对应（想象 CocoaPods 自动生成的注册表） | flutter 工具生成，把 pubspec 里所有插件的原生侧注册进引擎 | 手动删改会被下次构建覆盖 |
| `Generated.xcconfig` | 就是 xcconfig | 把 Flutter SDK 路径/构建模式接进 Xcode 构建 | 没跑过一次 flutter 命令前它不存在，纯 Xcode 构建会失败 |
| `FlutterActivity` (Android) | 对应 iOS 侧 FlutterViewController + 宿主 | `MainActivity : FlutterActivity()` 一行就够 | Android 的"页面"是 Activity，没有 VC 概念 |
| `AndroidManifest.xml` | `Info.plist` | 权限声明、显示名、入口 Activity | 权限要双端各声明一次（L4 会真踩到） |

## 四、双端对照

| 角色 | iOS | Android |
|---|---|---|
| App 入口 | `AppDelegate.swift`（+ `SceneDelegate.swift`） | `AndroidManifest.xml` 声明的入口 Activity |
| Flutter 宿主 | `FlutterViewController`（模板里由 storyboard 装配） | `MainActivity : FlutterActivity()` |
| 原生配置 | `Info.plist` | `AndroidManifest.xml` |
| 构建接线 | `Flutter/Generated.xcconfig` | `android/settings.gradle.kts` 里 include flutter 的 gradle 插件 |
| 依赖管理 | CocoaPods（`Podfile`，装了插件后出现） | Gradle（`build.gradle.kts`） |
