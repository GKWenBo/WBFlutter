## Q1. Dart 的 `int` 传到 Swift、Kotlin 各变成什么类型？为什么 Android 侧 `map["ts"] as Int` 有时会崩、正确写法是什么？

**类型落点：**

| 值 | Swift (iOS) | Kotlin (Android) |
|---|---|---|
| Dart `int` | 一律 `NSNumber`，取 `.intValue` 得 `Int`（64 位平台 Int 是 64 位，安全） | 小值（≤ int32）是 `Integer`(`Int`)；**大值（> int32）是 `Long`** |

**为什么"有时"崩：** StandardMessageCodec 编码时按值大小选 int32 / int64 打包。到 Kotlin 侧就分裂成两种对象——`3` 是 `Integer`，`1700000000000`（毫秒时间戳，超 int32）是 `Long`。`map["ts"] as Int` 对 `Long` 对象是失败强转 → **`ClassCastException`**；对小整数才不崩。所以崩不崩取决于**值大小**，这是最阴的地方（本地小数据测着好好的，线上大 id/时间戳一来就炸）。

**正确写法：** `(map["ts"] as Number).toLong()` —— 先当 `Number`（`Integer`/`Long` 的共同父类），再 `.toLong()`，无论底层哪种都安全。iOS 侧因为统一是 `NSNumber`，`.intValue`/`.int64Value` 没这个坑。

## Q2. 原生约定回 `int` 却回了 `double`，Dart 用 `invokeMethod<int>` 接会发生什么？崩在哪一端、什么异常？跟 L1 `getSystemUptime` 用 `double` 有什么关系？

- codec **不做隐式数字转换**。原生回 `3.0`(double)，Dart 侧 `invokeMethod<int>` 在把结果 `as int` 时抛 **`TypeError`**（`type 'double' is not a subtype of type 'int'`）。
- **崩在 Dart 端**，不是原生端——原生已经成功把 double 发回来了，是 Dart 侧的类型转换失败。见测试 `类型坑：原生返回 double 却用 <int> 接`（[l2_analytics_test.dart](../../../app/test/l2_analytics_test.dart)）。
- **与 L1 的关系：** L1 `getSystemUptime` 原生用的是 `ProcessInfo.processInfo.systemUptime`，类型是 `Double`(`NSTimeInterval`)。若 Dart 侧用 `invokeMethod<int>` 接必崩，所以必须 `fetchUptime() -> Future<double>` 用 `invokeMethod<double>`。L1 那个"为什么是 double"的伏笔，答案就是本课的坑 1。

## Q3. 要把一张缩略图的字节传给原生，用什么类型？换成 base64 `String` 传有什么代价？

- **用 `Uint8List`**（→ iOS `FlutterStandardTypedData`、Android `ByteArray`）。走 codec 的 typed-data 分支，按字节**零拷贝**打包。见 `AnalyticsBridge.uploadRawLog`（[analytics_bridge.dart](../../../app/lib/lessons/l2/analytics_bridge.dart)）。
- **换 base64 String 的代价：** ① 体积膨胀约 **33%**（每 3 字节 → 4 字符）；② 两端各多一次 base64 编/解码的 CPU；③ 字符串还要再走一层 UTF-8 编码。对大图 / 高频传输是明显浪费——所以规矩是二进制就用 `Uint8List`，别 base64。

## Q4. `fetchLoggedEvents()` 里 `invokeListMethod` 返回的 List，每个元素的静态类型是什么？为什么还要对每个元素 `.cast<String,Object?>()`？

- `invokeListMethod<Object?>()` 返回 `List<Object?>?`：它只收窄了**顶层是 List**，**每个元素的静态类型是 `Object?`**（运行时其实是 `Map<Object?, Object?>`）。
- **为什么还要 `.cast`：** 每个元素运行时是 `Map<Object?, Object?>`（键值都是 `Object?`）。要喂给 `AnalyticsEvent.fromMap(Map<String, Object?>)`，不能直接 `as Map<String, Object?>`——那是对"键值为 Object? 的 Map"做失败的整体强转。得用 `.cast<String, Object?>()` 逐项做运行时类型检查再收窄。这正是本课**坑 3**（`Map<Object?,Object?>` 不能直接当强类型 Map 用）的实战应用。见 [analytics_bridge.dart](../../../app/lib/lessons/l2/analytics_bridge.dart) `fetchLoggedEvents`。
