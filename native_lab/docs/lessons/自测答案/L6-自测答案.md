# L6 自测答案

**1. `viewType` 在三端各出现在哪里？写不一致会怎样？**

- Dart：`const kNativeViewType = 'com.wenbo.native_lab/native_view'`，传给 `UiKitView/AndroidView(viewType:)`。
- iOS：`registrar.register(factory, withId: "com.wenbo.native_lab/native_view")`。
- Android：`registry.registerViewFactory("com.wenbo.native_lab/native_view", factory)`。

三端靠这个字符串对齐。若不一致：Flutter 按它去找工厂，**原生没有注册同名工厂 → 视图创建失败**（iOS 报找不到 platform view factory / Android 抛异常或空白）。和前几课"channel 名打错运行期才崩"同源——字符串魔值的老问题，所以要抽成常量、三端逐字符核对。

**2. 为什么每实例一条通道、而不是应用级单例？viewId 从哪来？**

一个页面可能同时嵌**多个同类原生视图**（两张地图、三个网页）。若共用一条应用级 channel，"reload 哪一个"就无法寻址。所以每个实例的通道名带 `viewId`（`native_view_<viewId>`），一一对应。`viewId` 由框架在创建视图时分配，Dart 侧从 `onPlatformViewCreated: (int id)` 拿到，据此为该实例建一个 `NativeViewController`。前五课都是"一个 App 一个桥"，所以单例就够；这是 PlatformView 特有的多实例诉求。

**3. `creationParams` 和"创建后的方法通道"分别解决什么？codec 为何要对齐？**

- `creationParams`：**创建那一刻**把初始状态一次性传给原生（iOS 初始地图区域、Android 初始 URL）。单向、只在创建时。
- 方法通道：视图**创建之后**持续控制它（切地图类型、reload、loadUrl），可多次、可带返回。
- codec 对齐：`creationParams` 要跨语言序列化。Dart 用 `StandardMessageCodec` 编码，原生 Factory 的 `createArgsCodec()` 必须用**同一套** codec（iOS `FlutterStandardMessageCodec.sharedInstance()`、Android `StandardMessageCodec.INSTANCE`）才能正确解出。不一致就是乱码/解析失败（对照 L2 的类型映射：编解码两端必须同一套规则）。

**4. 性能代价来自哪？据此何时用、何时别用？**

原生视图不在 Flutter 的渲染管线里，要把它的内容**合成进 Flutter 的图层树**——涉及纹理拷贝、额外图层、有时还要平台线程与 raster 线程同步。这比一个纯 Flutter widget 贵得多。结论：**能用 Flutter 画的（按钮、卡片、列表、自绘图形）就别嵌原生视图**；只有当你需要一个 **Flutter 造不出的真实原生 SDK 视图**（地图、网页、相机预览、原生广告横幅）时，才付这个代价用 PlatformView。

**5. L6 和 L4 页面级混合怎么选？**

- L4：整屏交给原生页，用户操作完（扫码/拍照/原生登录）就退回 Flutter。**一次性任务**，原生页期间 Flutter 不可见，代价低。
- L6：原生视图作为 widget **长驻**在 Flutter 布局里，和 Flutter 控件**同屏共存**。适合"需要边看原生视图边用 Flutter UI"（地图上叠 Flutter 面板、网页上方放 Flutter 工具栏）。代价高。
- 一句话：**要不要和 Flutter 同屏共存**——要就 L6，不要（整屏接管、拿完即走）就 L4。

**6. 为什么不用 widget test 验证地图/网页真显示了？那测什么？**

widget test 跑在无原生宿主的纯 Dart 环境，`MKMapView`/`WebView` 根本不会被真正创建和渲染（我们还得 mock 掉 `flutter/platform_views` 系统通道免得抛错）。所以"地图真的画出来了吗"只能靠**模拟器实跑 + 截图**验证（过关四件套里的第 1 项）。widget test 能测的是 **Dart 侧逻辑**：① 控制器把按钮动作正确编码成 method call；② `NativePlatformView` 按平台产出正确的 `UiKitView`/`AndroidView`；③ 页面能 build。原生渲染归模拟器验证，Dart 逻辑归单测——各管一段。
